## Architectural Decisions Made This Session

### ADR-level: Environment variable loading strategy
**Decision**: `packages/db/src/client.ts` now auto-loads `.env` by walking up the directory tree from the compiled file's location. This ensures all consumers of `@byrdos/db` (API, workers, scheduler) pick up `DATABASE_URL` without per-service configuration.

**Rationale**: Every service imports `@byrdos/db`, which triggers `postgres(DATABASE_URL)` at module-load time. Without env loading, `postgres(undefined)` defaults to OS user/password. Fixing it once in the shared client eliminates per-service boilerplate.

**Trade-off**: Slightly less explicit than per-service env loading, but eliminates an entire class of silent connection failures. In production, `DATABASE_URL` is already set in the environment, so the walk-up is a no-op.

### Config fix: Turbo concurrency type
**Decision**: Changed `turbo.json` `"concurrency": 20` (number) to `"concurrency": "20"` (string). Turbo 2.10.5 requires string.
