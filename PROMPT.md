We are building byrdOS, an AI-first personal operating system that helps users understand, automate, and optimize every aspect of their financial life.

The first production integration is Varo Bank, but the architecture must be provider-agnostic so additional banking institutions, investment platforms, budgeting tools, payroll providers, and financial APIs can be added later with minimal changes.

The goal is to design a clean, scalable, enterprise-grade architecture before writing implementation code.

For this planning phase, produce:

• Overall system architecture
• Monorepo structure
• Package responsibilities
• Application boundaries
• Domain-driven design
• External integration strategy
• Authentication flow
• OAuth lifecycle
• API abstraction layer
• Synchronization pipeline
• Background job architecture
• Event system
• Database schema
• Entity relationships
• Service layer
• Repository layer
• Error handling strategy
• Retry strategy
• Caching strategy
• Logging strategy
• Security considerations
• Testing strategy
• Deployment strategy
• CI/CD recommendations
• Future scalability considerations

For the frontend, define:

• Route structure
• Dashboard layout
• Shared components
• State management
• Data fetching strategy
• Loading/error states
• Mobile responsiveness
• Design system recommendations

For the backend, define:

• Modules
• Services
• Controllers
• Workers
• Scheduled jobs
• Queues
• Domain models
• Interfaces
• Dependency injection strategy

For developer experience, define:

• Folder structure
• Naming conventions
• Code standards
• Agent responsibilities
• Documentation strategy
• Graphify knowledge graph structure
• Recommended prompts for each specialized agent

Visualize:

• System architecture
• Package dependency graph
• Entity relationship diagram
• OAuth flow
• Data synchronization flow
• Event flow
• Request lifecycle

The output should be implementation-ready documentation suitable for a senior engineering team.

Do not begin coding.

Instead, produce an engineering plan, architecture documentation, diagrams, implementation roadmap, and prioritized milestones that specialized coding agents can execute independently with minimal context.
